The SWIGB Project Decision Tree
A repeatable, evidence-based method for ranking any project — impact and execution first, money second.

Project Identification

Name the project you are evaluating. This label will carry through every step of the decision tree.

The theory of change, in one breath: We gate every project against our non-negotiables, score it on impact and execution, and only then run the numbers. The result is a clean ranked pipeline where flagship projects are obvious, quick wins fund the mission, and nothing gets built that we would not be proud of long after we are gone.

Step 0 — The Four Knockout Gates

Pass/fail. These encode the non-negotiables. A project that fails any gate is not scored — it is declined, parked, or restructured.

Gate A — Mission Fit
Does it pass the three gold-star rules?
Meaningful work, worth doing, and a benefit that outlasts us. Free of "silly work." A clear theory of change can be stated in three sentences.
If it fails: Decline. No score.
Gate B — Control Lock
Can SWIGB hold 51% managing, non-dilutable?
Structure permits SWIGB Enterprises 51% managing position with no dilution without written consent.
If it fails: Restructure, or decline.
Gate C — Capital Posture
Can we participate at or near $0 sponsor equity?
Returns come from fees, promote, IP, and post-flip ownership — not project equity at risk.
If it fails: Exception review by managing partner only.
Gate D — Disclosure Safety
Is the idea protected and the disclosure tier correct?
Core method is a registered trade secret and/or behind an executed MNDA; nothing above Tier 1 has been shared without one.
If it fails: Park until protected. No disclosure.
One or more gates failed. The project cannot proceed to scoring. Address the failures or restructure.

Why impact and execution rank above money: A project that scores high on impact and execution but only middling on returns is still worth pursuing — we can usually restructure the money. Ranking on impact and execution first protects the mission from being quietly steered by a spreadsheet.
